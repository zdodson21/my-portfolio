Between ​May ​- ​August ​of ​2024 I ​created ​numerous ​web ​components ​for ​HAXTheWeb ​(now ​HAXLabs). These ​web ​components ​were ​utilized ​to ​create ​one ​big ​component, ​named ​chat-agent. ​The goal ​of ​this ​project ​was ​to ​create ​a ​chat ​agent ​for ​websites ​created ​with ​HAXcms that ​could essentially act ​as a ​virtual ​teacher, ​using ​website ​content ​and ​HAX ​knowledge ​(for ​developers) as context ​to ​answer ​user ​questions. ​Under the hood, a "store" is used to house anything that is needed across the various sub-components of the overhead chat-agent component. This includes global variables and functions. The code for this store can be viewed here.

chat-agent ​features ​will ​be ​described ​below ​next ​to ​web components they belong to.

Chat Agent

The ​overhead ​chat-agent web ​component, ​housing ​numerous other ​web ​components ​inside ​of ​it. Overall ​this component ​doesn't ​have ​much ​functionality, but ​it ​does ​respond ​to ​visual ​changes ​set ​by ​other ​components:

  • Handles ​full ​screen ​functionality, ​changing ​the ​appearance ​of ​the ​chat agent ​to ​take ​up ​100% of ​the ​available ​vertical ​screen ​space.
  • Shows ​and ​hides ​the ​chat interface, while ​the ​chat button ​will ​always ​be ​visible ​if the ​interface ​is ​not ​visible. ​The ​chat button ​will ​hide ​if ​the ​chat interface is ​set ​to ​take ​up ​the ​full vertical ​screen ​space.


Chat ​Button

Handles ​the ​opening and ​closing ​of ​the ​chat-interface ​component. ​Because this ​button is ​primarily ​made ​of ​the ​<div></div> ​HTML tag, ​CSS ​was ​used ​to ​make ​the ​button ​animate like ​a ​button ​to ​improve ​end-user usability ​and ​understanding.


Chat ​Control ​Bar

Handles numerous ​functions, ​in ​order: ​Downloading ​chat ​log, ​restarting ​chat session, ​toggling ​data ​collection, ​toggling ​developer mode, ​toggle ​full ​screen ​mode, ​toggling visibility ​(when ​in ​full ​screen ​mode).


Chat ​Developer ​Panel


Provides ​various ​developer ​features, ​such ​as ​printing ​the ​chat ​log ​to ​the ​browser console, ​downloading ​the ​chat ​log ​as ​a ​.json ​file, ​and ​switching both ​the ​back-end ​AI ​model ​and ​the ​context ​used ​to ​answer ​questions.


Chat ​Input

This ​component ​is ​where ​users ​will ​type ​their ​messages, ​and ​send ​them to ​the ​back-end ​for ​response. ​Users ​can ​also ​utilize ​the ​arrow ​buttons ​on ​the ​right ​side ​to ​view ​and ​submit ​previous ​messages. ​Keyboard navigation included for ​more ​fluid usage ​and ​accessibility.


Chat ​Interface

Chat ​Interface ​is ​the ​main ​window ​that ​will ​be ​used to ​interact with ​the ​chat ​agent ​and ​its many ​features. ​It ​houses ​the ​following ​components ​within ​it, ​handling ​many ​styling ​aspects ​of the ​child ​components:

  • Chat ​Control ​Bar
  • Chat ​Developer ​Panel ​(if ​enabled ​in ​control ​bar)
  • Chat ​Message ​(displayed ​from ​chat ​log ​using ​array ​mapping)
  • Chat ​Input


Chat ​Message

Chat ​message ​will ​display ​the ​message ​it ​is ​assigned from ​the ​chat ​log, ​while ​ensuring ​proper ​formatting ​according ​to ​who ​sent ​the ​message, ​using ​a ​hat ​image ​for ​Merlin ​(the name for ​the ​AI), ​or ​an ​RPG ​character for the ​user ​(RPG character is ​generated ​using ​the user's ​username). ​If ​the ​message ​is ​sent ​by ​Merlin, chat ​suggestions will ​be ​displayed ​as ​well.


Chat ​Suggestion

Provides ​suggested ​prompts ​to ​the ​user. ​When ​one is ​selected, the ​selected ​suggestion ​will ​turn ​green, ​while the ​rest ​will ​turn ​red. ​Selected ​suggestion ​is ​automatically ​sent as ​a ​message ​to ​the ​AI ​back-end.


Chat ​Agent ​(Full Screen ​Mode)

Full ​screen ​mode ​provides the ​following changes:

  • Hides ​the ​chat-button ​component ​when ​visible, ​can ​be ​hidden ​by ​pressing ​the ​"minus" ​button ​on the ​right ​side ​of ​the ​chat ​control ​bar.
  • More ​visible ​messages on ​screen
  • Conforms ​to ​the ​right ​side ​of ​the ​screen while ​forcing ​website ​content ​to ​appear ​to ​the ​left ​of ​it. Improves ​visibility of ​website ​content ​by ​no ​longer ​covering ​the content ​with ​a ​button ​and ​interface.
  • CSS ​media ​queries ensure ​full ​screen ​mode ​remains visually ​appealing ​on ​various screen ​and ​window sizes.


Chat ​Log


While ​not ​a ​component, ​the ​chat ​log ​(located ​in ​the ​store) plays ​a ​vital ​role ​in ​the ​functionality ​of ​chat-agent. ​It ​enables ​message ​rendering, message ​history, chat ​saving, ​and ​in ​theory ​could ​be ​used ​as ​context ​for ​an AI ​back-end ​to ​answer ​questions.
